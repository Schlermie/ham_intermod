How to run HamIM

HamIM is a webapp to compute intermodulation distortion (IMD) conflicts, given a list of radio frequencies to be used at an event.

The only input is a CSV file with two columns listing the frequencies planned for your event. The column headers must include "RxFreq" and "TxFreq" for your receiving and transmitting frequencies, respectively. For repeater channels, the TxFreq column can use the shorthand notation "+" or "-" to indicate your repeater offset direction, and HamIM will compute the transmit frequency. For simplex channels, the TxFreq column can remain blank. Any other columns will be ignored. In the example below, the column labeled "Label" will be ignored.
